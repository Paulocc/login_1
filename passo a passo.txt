Primeiramente, garanta que o node.js esteja instalado
para verificar, abra o powershell e digite: 
node -v
Caso apareça a versão, significa que está instalado.
Digite:
npm -v
Caso apareça a versão, o gerenciador de pacotes do node também está funcionando. Caso
apareça algum erro, abra o PowerShell como administrador, e execute o seguinte comando:
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
Pressione a tecla A e de enter
Com isso, novamente teste o npm -v, desta vez a versão irá aparecer sem problemas.

Com isso pronto, no vscode, crie uma nova pasta para iniciar o projeto
Abra o terminal no vscode (ctrl + shift + ')
Digite os seguintes comandos

npm init -y
npm install express bcrypt jsonwebtoken cookie-parser
npm install @prisma/client@6 prisma@6 --save-dev
npx prisma init --datasource-provider sqlite

Com isso os pacotes necessários para executar o projeto serão instalados.
Alguns arquivos serão criados com isso, um deles é a pasta prisma
Abra essa pasta e modifique o arquivo schema.prisma com o arquivo de mesmo nome neste repositório.

Execute:
npx prisma migrate dev --name init

crie o arquivo seed.js e server.js na raiz do projeto
Novamente, modifique esses arquivos criados copiando os arquivos de mesmo nome neste repositório.
execute: 

node seed.js

Se rodar sem problemas, uma mensagem irá aparecer indicando sucesso.
Após isso, crie duas pastas, uma public e uma private
crie o arquivo index.html na pasta public
crie os arquivos user.html e admin.html na pasta private

Modifique esses arquivos copiando os arquivos de mesmo nome neste repositório.

execute:

node server.js

Se tudo estiver certo, pelo navegador acesse: localhost:3000

Assim irá aparecer a tela de login, teste os logins de administrador e de usuário
